# 2026-07-02 작업 기록

## 구현한 것

- `app/ai/embeddings/openai_client.py` — OpenAI `text-embedding-3-small` 얇은 래퍼 (`embed_text`/`embed_texts`)
- `app/ai/embeddings/activity_embeddings.py` — embedding이 비어있는 Activity를 찾아 title+category+description으로 임베딩 생성, 100개씩 배치 처리
- `app/ai/recommendations/extracurricular_recommender.py` — 사용자 프로필(department + major + career_goal) 임베딩과 Activity 임베딩의 코사인 유사도 계산
  - `final_score = similarity_score * career_weight * recency_weight`
  - career_weight: career_goal 있으면 1.2배, 없으면 1.0배
  - recency_weight: 게시일 기준 90일 선형 감쇠 (최근 1.0 → 90일 0.5)
  - 마감 지난 활동/90일 이전 게시물은 후보에서 제외, 결과는 `UserActivityRecommendation`에 upsert
- `app/api/activities.py` — `GET /activities/recommendations/{user_id}` 추천 그리드 API
  - 캐시된 추천이 없으면 즉시 계산 후 반환 (최초 요청 시 지연 있음)
  - 응답에 d-day, 카테고리, 추천 점수(%) 포함
- `app/core/scheduler.py` — 자정 크롤 직후 임베딩 생성 → 전체 사용자 추천 재계산까지 이어서 실행하도록 확장
- `app/main.py` — activities 라우터 등록
- `requirements.txt` — `openai` 패키지 명시 추가 (기존엔 langchain-openai 통해 간접 설치됨)

## 고친 것

- `app/ai/embeddings/activity_embeddings.py`: 제목이 빈 문자열인 Activity(크롤러 파싱 실패 케이스 1건 확인)는 OpenAI 임베딩 API가 400을 반환해 배치 조회에서 제외 처리

## 막혔던 부분 / 삽질

1. 인증 시스템이 아직 없어서(로그인/JWT 미구현) 추천 API가 `user_id`를 경로 파라미터로 직접 받음 → 로그인 붙으면 `get_current_user` 의존성으로 교체 필요
2. job 게시판 출처 중 제목이 빈 문자열로 크롤된 공지 1건 발견 → 원인(파싱 실패 vs 실제 빈 제목)은 미조사, 우선 임베딩 배치에서 스킵만 처리
3. `UserActivityRecommendation.user_id`/`activity_id`에 FK 제약이 없어서 테스트 유저 삭제 시 추천 레코드가 고아로 남음 → 수동 정리함, FK 추가 여부는 논의 필요
