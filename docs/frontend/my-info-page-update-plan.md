# 내 정보 페이지 수정 계획

## 목표

내 정보 페이지에서 사용자가 One-Stop(학생지원시스템) 계정으로 교과 정보를 동기화하면,
화면의 졸업요건과 학기별 성적이 실제 데이터로 갱신되도록 한다. 프로필 진입 동선도
왼쪽 하단의 사용자 영역으로 통일한다.

## 1. 교과 활동 불러오기

### 1-1. 입력 항목 변경

- 기존: `학번` + `이름`
- 변경: `학번` + `학지시 비밀번호`
- 비밀번호 입력칸은 `type="password"`로 표시한다.
- 두 입력값은 `POST /me/portal-sync` 요청 본문의 `login_id`, `password`로 전송한다.
- 요청이 끝난 뒤 비밀번호 입력값은 프론트 화면 상태에서 즉시 비운다.

### 1-2. 입력 예시 제거

- 학번 입력칸의 placeholder 예시(`예: 2023662247`)는 유지한다.
- 비밀번호 입력칸에는 placeholder 예시 문구를 넣지 않는다.

### 1-3. 동기화 중/완료 상태

1. 사용자가 `교과 활동 불러오기`를 누른다.
2. 버튼을 비활성화하고, 배경을 흰색으로 변경한 뒤 `불러오는 중...` 상태를 표시한다.
3. `POST /me/portal-sync`가 성공하면 `GET /me/graduation`을 호출한다.
4. 동기화 응답의 수강 과목과 졸업요건 응답을 화면 상태에 반영한다.
5. 최신 사용자 학적 정보를 반영하기 위해 `GET /auth/me`도 다시 호출한다.
6. 데이터 반영이 완료되면 페이지를 새로고침한다.
7. One-Stop 로그인 실패, 네트워크 오류 등은 입력 폼 아래에 오류 메시지로 표시하고 버튼을
   다시 활성화한다.

### 1-4. 새로고침 뒤 학기별 성적 표시를 위한 선행 조건

- 현재 `POST /me/portal-sync`는 수강 과목 목록을 응답으로 주지만, 새로고침 후 저장된
  성적 목록을 다시 조회하는 `GET` API는 없다.
- 따라서 졸업요건은 새로고침 뒤에도 `GET /me/graduation`으로 다시 표시할 수 있지만,
  학기별 성적은 아래 두 방식 중 하나가 필요하다.
  - 권장: 백엔드에 로그인 사용자용 성적 조회 API(예: `GET /me/course-records`)를 추가한다.
  - 임시 데모: 동기화 응답의 수강 과목을 `sessionStorage`에 저장한 뒤 새로고침 후 읽는다.
- 이번 프론트 작업을 시작하기 전에 팀 백엔드 담당자에게 성적 조회 API 추가 가능 여부를
  확인한다. API가 준비되면 권장 방식을 사용한다.

## 2. 프로필 카드 버튼

### 2-1. 자동 편집 삭제

- 프로필 카드의 `자동 편집` 버튼을 삭제한다.
- 교과 정보 자동 동기화는 페이지 상단의 `교과 활동 불러오기` 기능으로만 제공한다.

### 2-2. 편집하기 / 저장하기

- 이번 변경에서는 기존 `편집하기`, `저장하기` 버튼의 위치와 디자인을 유지한다.
- 비교과 활동, 자격증, 어학 성적을 직접 편집하고 저장하는 기능은 별도 작업으로 진행한다.
- 졸업요건, 학부·전공, 성적처럼 One-Stop에서 가져오는 공식 학사 정보는 직접 편집 대상으로
  두지 않는다.

## 3. 프로필 진입 동선

### 3-1. 상단 내 정보 버튼 삭제

- 우측 상단의 `내 정보` 버튼을 삭제한다.

### 3-2. 왼쪽 하단 미니 프로필 전체 클릭 처리

- 왼쪽 하단의 아바타, 이름, `나의 프로필 보기` 문구를 포함한 영역 전체를 클릭 가능하게
  변경한다.
- 클릭 시 `/info`로 이동한다.
- 키보드로도 접근할 수 있도록 링크 또는 버튼 의미를 유지한다.
- 현재 페이지가 `/info`여도 클릭 영역의 모양과 기능은 일관되게 유지한다.

## API 및 화면 데이터 연결

| 화면 데이터 | API | 사용 방식 |
| --- | --- | --- |
| 학번, 이름, 학부, 전공, 학적 상태 | `GET /auth/me` | 동기화 전 최초 표시 및 동기화 후 재조회 |
| One-Stop 교과 정보 동기화 | `POST /me/portal-sync` | 학번과 학지시 비밀번호를 전송 |
| 졸업요건별 이수/필요/남은 학점 | `GET /me/graduation` | 전공기초·전공필수·교양 등 진행 막대 표시 |
| 학기별 이수 과목 | 성적 조회 API 추가 필요 | `POST /me/portal-sync` 응답은 동기화 직후에만 사용 가능 |

## 완료 기준

- 학번과 학지시 비밀번호만 입력할 수 있다.
- 동기화 중에는 버튼이 흰색의 로딩 상태로 보이며 중복 클릭할 수 없다.
- 동기화가 끝난 뒤 졸업요건이 서버 데이터로 표시된다.
- 성적 조회 API가 준비되면 새로고침 뒤에도 학기별 과목이 서버 데이터로 표시된다.
- `자동 편집` 버튼과 상단 `내 정보` 버튼이 없다.
- 왼쪽 하단의 미니 프로필 전체를 누르면 내 정보 페이지로 이동한다.
- 로그인하지 않았거나 One-Stop 인증에 실패한 경우에도 사용자가 원인을 확인할 수 있다.

## 이번 작업에서 제외

- 비교과 활동, 자격증, 어학 성적의 편집·저장 기능
- 학사정보를 기반으로 한 학년/현재 학기 자동 계산 규칙 확정
- AI 자동 편집 및 AI 추천 기능
- 복수전공·부전공의 세부 졸업요건 판정

## 비교과 활동 링크

- 각 비교과 활동의 기존 문구(`활동 기록 추가/보기`, `공고 링크 추가`, `GitHub 링크 추가`)를
  인터랙션 버튼으로 사용한다.
- 버튼을 누르면 링크 입력칸과 확인·취소 버튼을 표시한다.
- 프로토콜을 생략하면 `https://`를 보완하고 올바른 웹 주소인지 검사한다.
- 확인하면 등록된 링크를 목록에 추가하고, 기존 문구의 추가 버튼은 목록 아래에 유지한다.
- 목업 링크는 활동별로 여러 개 등록할 수 있으며, 새로고침에는 유지되지만 탭을 닫으면
  초기화되는 브라우저 세션 저장소에 보관한다.

## 로컬 목업 모드

- 로컬 `.env`에서 `VITE_USE_MOCK_STUDENT_DATA=true`로 설정하면 백엔드 없이 프론트
  동작을 확인할 수 있다.
- 임의의 학번과 비밀번호를 입력하면 약 1.2초 동안 로딩 상태를 표시한 뒤 테스트 학생,
  학기별 과목, 졸업요건 데이터를 저장하고 페이지를 새로고침한다.
- 목업 학번은 숫자나 길이 형식을 검사하지 않으며, 비어 있지 않은 임의의 문자열을 허용한다.
- 목업 학기별 성적은 과목명, 이수구분, 학점, 임의 성적을 표로 표시한다.
- 목업 모드는 Vite 개발 환경에서만 활성화되므로 배포 빌드에서는 사용되지 않는다.
- 목업 확인은 실제 One-Stop 로그인 및 백엔드 API 연결 테스트를 대체하지 않는다.
