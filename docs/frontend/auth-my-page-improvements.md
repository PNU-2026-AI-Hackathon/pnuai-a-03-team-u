## 로그인 및 회원가입

### 인증 화면 진입 흐름

- 비로그인 사용자가 서비스에 접속하면 로그인 페이지를 먼저 표시한다.
- 인증이 필요한 화면은 보호 라우트로 감싸고, 로그인 완료 후 홈으로 이동한다.
- 로그아웃 시 로컬·세션 저장소의 실제 토큰과 목업 인증 정보를 함께 정리한다.

### 목업 로그인

- 개발 환경에서 목업 인증이 활성화되면 임의의 아이디와 비밀번호로 로그인할 수 있다.
- 목업 로그인도 실제 로그인과 같은 `AuthContext` 흐름을 거쳐 사용자 정보를 제공한다.
- 로그인 유지 체크 시 `localStorage`, 미체크 시 `sessionStorage`에 인증 상태를 저장한다.
- API 요청 인터셉터가 두 저장소의 실제 액세스 토큰을 모두 확인한다.

### 화면 정리

- 목업 로그인 입력창의 안내 문구를 제거했다.
- 회원가입의 학번·학과 옆 `선택` 문구를 제거했다.
- 동작하지 않던 부전공·복수전공 `추가` 버튼을 제거했다.
- 기존 회원가입 API 호출과 성공 후 로그인 화면 전환은 유지한다.

## 내 정보 페이지

### 교과 활동 동기화

- 입력 항목을 `학번 + 이름`에서 `학번 + 학지시 비밀번호`로 변경했다.
- 학번 예시는 유지하고 비밀번호 예시는 제거했다.
- 동기화 중 버튼 로딩 상태를 표시하고, 완료 후 페이지를 다시 불러와 교과·졸업 정보를 반영한다.
- 개발 환경에서는 임의 학번과 비밀번호로 목업 교과 데이터를 불러올 수 있다.

### 프로필 편집

- 이름, 학부, 전공, 학년을 편집하고 저장할 수 있다.
- 학부와 전공을 줄바꿈해 구분하여 표시한다.
- 학부를 포함한 수정 결과는 브라우저 세션에서 새로고침 후에도 유지한다.
- 프로필의 자동 편집 버튼과 상단의 중복 내 정보 버튼을 제거했다.
- 사이드바 하단의 `나의 프로필 보기` 전체 영역을 링크로 동작하게 했다.

### 졸업요건 편집

- 총 이수학점, 졸업 기준학점과 이수구분별 이수·기준학점을 수정할 수 있다.
- 숫자 입력창의 증감 화살표를 숨겨 직접 입력 방식으로 정리했다.
- 하위 이수구분의 이수학점·기준학점 합계를 편집 화면에 표시한다.
- 졸업요건을 수정한 경우 상단 합계와 하위 항목 합계가 일치하지 않으면 저장을 차단하고 오류를 안내한다.
- 이름이나 성적만 수정할 때는 기존 목업 졸업요건의 합계 차이로 저장이 차단되지 않는다.

### 학기별 성적

- 4.5 만점 기준 전체 총평점·전공평점과 학기별 총평점·전공평점을 표시한다.
- 학점 가중 평균을 사용하며 `P`, `S` 과목은 평점 계산에서 제외한다.
- 편집 모드에서 기존 과목의 성적을 변경할 수 있다.
- 수강 과목의 과목명, 이수구분, 학점, 연도, 학기, 성적을 입력해 추가할 수 있다.
- 편집 중인 과목을 삭제할 수 있으며, 추가·수정·삭제 결과를 평점에 즉시 반영한다.

### 비교과·자격증·어학성적

- 비교과 활동, 자격증, 어학성적의 추가·수정·삭제 화면을 구현했다.
- 삭제 전 확인 UI와 API 오류 안내를 제공한다.
- 실제 환경에서는 `/me/activities`, `/me/certifications`, `/me/language-scores` API를 사용한다.
- 목업 환경에서는 동일한 조작을 `sessionStorage` 기반 데이터로 검증할 수 있다.
- 비교과 링크 추가 문구를 `활동 기록 추가`로 통일하고 입력 UI 크기를 정리했다.
- 다크모드의 세 추가 버튼을 어두운 청회색으로 조정해 시각적 강조를 낮췄다.

## 저장 및 API 경계

| 데이터 | 현재 저장 방식 |
| --- | --- |
| 실제 로그인 토큰 | 로그인 유지 선택에 따라 `localStorage` 또는 `sessionStorage` |
| 비교과·자격증·어학성적 | 실제 환경은 백엔드 CRUD API, 목업 환경은 `sessionStorage` |
| 이름·학부·전공·학년 | 현재 프론트엔드 `sessionStorage` |
| 직접 편집한 과목·성적 | 현재 프론트엔드 `sessionStorage` |
| 직접 편집한 졸업요건 | 현재 프론트엔드 `sessionStorage` |

프로필 기본정보, 공식 성적, 졸업요건을 서버에 영구 저장하려면 해당 수정 API가 추가된 뒤 현재 저장 함수를 API 호출로 교체해야 한다.
