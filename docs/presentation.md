# Plan-U · 발표 자료

> 부산대학교 학생을 위한 맞춤형 학업 설계 서비스

---

## 1. 문제 정의 및 사용자 요구

- 학생들은 **졸업요건, 성적, 어학·자격증, 비교과 활동** 등 자기 학업 데이터를 여러 시스템(학지시·수강신청·포털)에 흩어진 채로 관리한다.
- "내가 지금 졸업까지 뭐가 얼마나 남았는지", "다음 학기에 뭘 들어야 하는지"를 한눈에 보여주는 도구가 없다.
- 특히 **복수전공·부전공** 학생은 요건이 복잡해 스스로 로드맵을 짜기가 더 어렵다.
- → **학업 데이터를 한곳에 모으고, 그걸 근거로 앞으로의 계획까지 제안**해주는 서비스가 필요하다.

---

## 2. 해결 방안: 맞춤형 학업 설계 서비스

- 흩어진 학업 데이터를 **자동으로 수집·정규화**해 한 화면에 통합
- 수집된 데이터를 근거로 **졸업요건 충족 현황을 자동 계산**
- 여기에 **AI 상담 에이전트**를 얹어, 학생의 남은 요건·진로 목표를 반영한 **학기별 성장 로드맵**을 대화로 함께 설계

---

## 3. 핵심 기능

### ① 내 정보 확인

- 사용자는 **졸업요건 충족 현황**과 **자신의 이력**(성적, 어학성적, 자격증, 비교과 활동)을 한눈에 파악
- 기존엔 사용자가 직접 성적표를 입력해야 했으나 → **Playwright 기반 크롤링**으로 학지시 로그인 후 학적부·성적·졸업예정정보를 **DB에 자동 저장**
  - 포털 비밀번호는 **Fernet 암호화**해 저장(평문 저장 금지) — 개인정보 보호 정책 준수
- 크롤링한 이수내역을 학과별 졸업요건 기준학점과 **이수구분별로 대조**해 "남은 학점"을 자동 계산

### ② 성장 로드맵 추천 (AI 상담)

- 학생의 **졸업요건 남은 학점 · 이수내역 · 진로 목표**를 종합해 학기별 수강 로드맵을 **대화형으로 추천**
- **Human-in-the-loop 설계**: AI는 로드맵 DB를 직접 수정하지 않고 **"이 변경을 반영할까요?"** 형태로 제안 → **사용자가 승인한 항목만** 실제 반영 (생성·수정·삭제 모두 동일)
- 실제 개설 과목만 추천하도록 제한 — 존재하지 않는 과목명을 지어내지 않음

### ③ 시간표 추천

- 수강신청 기간에 맞춰, 확정된 로드맵을 **실제 개설 분반(요일·교시·교수)** 단위의 맞춤형 시간표로 구체화

### ④ 다중전공자 지원

- **부전공·복수전공** 학생에게도 각 전공(주전공/복수전공/부전공)별 요건을 반영한 맞춤형 로드맵·시간표 제공

## 4. 시스템 아키텍처 및 기술 스택

### 배포 구성

| 구분 | 스택 | 배포 |
|---|---|---|
| **Frontend** | React 19 + Vite + TypeScript, axios · JWT 인증 | **Vercel** |
| **Backend** | FastAPI + SQLAlchemy + Alembic | **Railway** |
| **Database** | Supabase (Managed Postgres + pgvector) | 관리형 |

- **DB 저장 대상**: 학사 계층(학교–학부/학과–전공–과목) · 졸업요건 · 성장 로드맵 · 이수내역

### AI 스택

- **성장 로드맵 상담 에이전트** (기본 `gpt-4o`)
  - **LangChain 기반 tool-calling agent**: 과목 검색 · 변경 제안(propose) · 승인 반영(confirm) · 졸업요건 현황 조회 도구를 LLM이 호출
  - **멀티 LLM 지원**: `init_chat_model` 추상화로 설정 한 줄만 바꿔 OpenAI/Anthropic/Google 프로바이더 교체 가능
  - **Human-in-the-loop**: DB 직접 수정 없이 항상 "제안 → 사용자 승인" 후에만 반영
- **정형 DB 기반 교육과정 검색**: 학과·학년·학기·이수구분 필터 + 교과목개요(description)까지 근거로 활용
  - `text-embedding-3-small` + **pgvector** 임베딩도 구축되어 있어, 필요 시 의미 기반 검색으로 확장 가능

> 시스템 구조 상세: `docs/system-architecture.svg` 참고

## 8. MVP 목표 및 향후 보완 계획

### MVP 목표

- **4가지 핵심 기능 구현**: 내 정보 확인 / 성장 로드맵 / 시간표 추천 / 다중전공자 지원
- **디자인 및 프론트엔드 구현**
- **DB 구축**: 2026 교육과정 기준 학과·과목 DB (우선 일부 학과 한정 세부 적용)
- **배포**: 프론트엔드 → Vercel, 백엔드 → Railway

### 향후 보완 계획

- **다양한 교육과정 연도 지원** (현재 2026 기준 → 입학년도별 요건 확대)
- **전 학과 DB 구축** (현재 일부 학과 → 전체 학과)
- **추천 활동 기능**: 사용자 특성에 따라 비교과·외부 활동 추천
- 로그인 식별자·API 계약 등 **프론트–백엔드 연동을 기능 단위로 조기 통합**하는 개발 프로세스 정착
