# 학교 계층(schools~majors) + 2026 교육과정(courses) Supabase 적재

2026-07-08 / 작성: 조현우 (백엔드·DB)

PR #45의 `schools → colleges → departments → majors` 계층과 `courses`를 실데이터로 채웠다.
반영 완료 상태(팀 공유 Supabase):

| 테이블 | 행 수 | 내용 |
|---|---:|---|
| schools | 1 | 부산대학교 |
| colleges | 16 | 현행 단과대 |
| departments | 109 | 학과/학부 (모집단위 포함) |
| majors | 36 | 학부/학과 하위 전공 |
| courses | 6,402 | 2026학년도 교육과정 (전공계열 6,345 + 공통 교양 57) |

## 데이터 소스와 재현 방법

- **소스는 전부 AIS(수강신청 시스템)다.** 학과명 표기·소속 배치·과목 정보 모두 AIS 응답을
  그대로 따랐다(추정·수기 입력 없음). K2Web 학과 사이트에 박혀 있는 교육과정 위젯이 실은
  전학교 프록시라서, 한 사이트의 위젯으로 모든 학과/전공/연도를 조회할 수 있다.
- 시드 파일(커밋됨): `backend/seeds/school_hierarchy_mapping.csv`(144개 단위 계층 매핑),
  `backend/seeds/ais_courses_2026.csv`(교육과정 7,651행 원본 스냅샷).
- 실행(멱등 — 몇 번을 다시 돌려도 안전):
  ```bash
  python -m scripts.seed_school_hierarchy      # 계층
  python -m scripts.import_courses_from_ais    # 교육과정
  ```

## 공통적으로 알아야 할 컨벤션 (전원 숙지)

1. **학과 단위 조회는 `major_id IS NULL` 필터가 필수다.** 전공을 가진 부모 학과 13곳
   (무용학과, 전기전자공학부, 정보컴퓨터공학부, 약학부 등)은 `department_id`만으로 조회하면
   하위 전공 데이터가 섞여 나온다. 졸업요건 판정을 포함한 모든 코드에 적용된다.
2. **`majors.name`에는 잘린 전공명만 있다** ("전기전자공학부 반도체공학전공"이 아니라
   "반도체공학전공"). `hierarchy.py`의 완전일치 get-or-create와 호환되게 하기 위함.
3. **교양 과목은 `department_id IS NULL`인 공통 행 1벌(57과목)이다.** 효원핵심/균형/창의·기초교양은
   전학교 공통이라 학과별로 중복 저장하지 않았다. 성적 매칭 시 교양은 이 행들과 매칭된다.
4. **`courses` 행 = "해당 학과/전공 교육과정의 과목 항목"이다.** 같은 과목(예: 공학미적분학(I))이
   여러 학과 교육과정에 있으면 행이 여러 개다. `course_code`가 비유니크 인덱스인 이유.
   멱등 키는 (department_id, major_id, course_code).
5. **`courses.year`는 연도가 아니라 "학년"이다** — 값: `1`~`6`, `전학년`.
   `semester` 값: `1`, `2`, `1,2`, `전학기`, `여름/겨울 계절·도약수업`. 6년제(약학·치의)와
   5년제(건축학과)가 있어 5·6학년 값이 정상 존재한다.
6. **`courses`는 "항상 최신 교육과정"만 유지한다** (연도별 스냅샷을 쌓지 않는다). 근거:
   ① courses는 plans/offerings/records가 참조하는 FK 허브라 버전화 부적합,
   ② 과거 학번의 이수구분 정확성은 성적표 원본(`student_course_records.category`)이 이미 담보,
   ③ 연도별 요건 변동은 `graduation_requirements.curriculum_year`가 담당.
   연도별 원본은 raw_data에 CSV로 보관하고, 매년 AIS를 재fetch해 upsert하면 된다.
7. **가운뎃점 표기는 AIS 원문 그대로다** — AIS 자체가 혼용한다(`조선·해양공학과`는 `·`,
   `관악.타악전공`·`한국.동아시아학전공`은 `.`). 임의로 통일하지 말 것: 포털/AIS 계열 입력과의
   완전일치 매칭이 우선이다.

## 특수 배치 결정 (팀 확인 이력)

- 자율전공형 4건(첨단IT/모빌리티/소재자율, 미래도시건축환경융합) = **공과대학 직속**, 1학년 전용
  모집단위(이후 실제 학과로 진학)
- 핀테크융합전공 = **경영대학 직속** (2학년부터 시작)
- 지능형헬스사이언스융합전공 = **자연과학대학 직속** (주관 수학과, 여러 학과 공동 운영)
- EES융합전공 = **학부대학 첨단융합학부의 전공** (나노과학기술대학 쪽 중복 코드 461910은 제외)
- 스마트시티전공 = **공과대학 직속** (AIS 편제 기준)
- 약학대학 = 약학부(385000, 1학년) → 2학년부터 약학전공(385001)/제약학전공(385002)
- 치의학과(401400) = 수집 인덱스에 누락돼 있어 수동 추가. 한의학과는 AIS 학부(UNIV) 트랙에
  없어(대학원 트랙 전용) 제외

## 이상 데이터 기록 (원본 자체의 특이사항)

1. **행정학과 `PA2700143`은 AIS 원본부터 교과목명이 공란이다** (2024~2026 모두 확인).
   실존 과목이라 버리지 않고 `(교과목명 미상: PA2700143)`으로 적재했다. UI에서 이 이름이
   보이면 이 케이스다.
2. **조선·해양공학과는 AIS에 동명 현행 코드가 2개다** (342100/342400). 342400은 교육과정이
   빈 껍데기고 342100이 실제 사용 코드다. 매핑은 342100 기준.
3. 교직과목 4건(교육학과 3, 특수교육과 1)은 AIS에 권장 이수 시점이 2개 표기돼 있다
   (예: 교육학개론 1-2 및 2-1). 적재 시 1행으로 수렴된다(마지막 표기 기준).
4. 기계공학부 "졸업논문"은 0학점이다 — P/F 과목으로 정상.
5. **AIS에는 폐지 마커 없는 옛 편제가 다수 현존한다** (상과대학, 국악학과, 건축학부,
   기계공학부 구트랙 전공, 사회환경시스템공학부 등 약 177개 단위). 시드에서 의도적으로
   제외했다 — 옛 소속 재학생이 포털 동기화되면 `hierarchy.py` get-or-create가 런타임에
   자동 생성한다(원설계 의도). 특히 **국악학과와 음악학과는 동명 전공(성악/작곡)이 있으니**
   전공명만으로 매칭하는 코드를 짜지 말 것.
6. 화공생명.환경공학부는 계층에는 있으나 courses가 0건이다 — 옛 통합 학부라서이고,
   현행 화공생명공학과(65과목)·환경공학과(68과목)가 커버한다.
7. 수집 조사 인덱스에는 조선·해양공학과가 가운뎃점 표기 차이(`·`/`・`)로 서로 다른
   프로그램 코드 2행으로 이중 등재돼 있었다 — 병합 처리했다. 표기 변형은 언제든 중복의
   원인이 될 수 있다.

## 추후 검토 필요

1. **Global Studies Program**: 소속 단과대 미확인(AIS 학부 트랙에도 없음) — 계층 미적재 상태.
2. **`courses`에 교육과정 연도 컬럼이 없다**: 2026 스냅샷만 반영되며, 연도별로 이수구분이
   바뀐 과목은 2026 기준으로 보인다. 학번별 정밀 판정이 필요해지면 courses를 버전화하지 말고
   별도 테이블(curriculum_entries류)을 설계할 것.
3. **`graduation_requirements`는 아직 비어 있다** — 다음 적재 대상 (규정집 rule_seq=84
   전학과 이수학점표 + 검증된 학과별 요건).
4. **포털 성적표 표기와 AIS 표기 차이 모니터링**: `건축학과` vs `건축학과(5년제)`류,
   가운뎃점 변형이 포털 동기화에서 중복 행을 만들 수 있다. 동기화 코드에 정규화 규칙 필요.
5. **빈 DB 부트스트랩이 현재 깨진다**: reset 마이그레이션(`f1a2b3c4d5e6`)이 삭제된
   `app.domains.activities`를 import해서 신규 로컬 셋업/CI에서 `alembic upgrade head`가
   실패한다(운영 Supabase는 무관). `scripts/seed_departments.py`도 구 스키마(name 단독
   unique) 전제라 재작성 또는 삭제가 필요하다.
6. **연 1회 갱신 루틴**: 새 학년도 교육과정 공개 시 AIS 재fetch → 스냅샷 CSV 교체 →
   `import_courses_from_ais` 재실행(멱등 upsert).
